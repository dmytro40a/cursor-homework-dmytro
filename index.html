• git config --global user.name blabla (задати ім'я користувача)
  • git config --global user.email blabal@bla (задати електронну адресу користувача)
  • git init (ініціалізація репозиторію)
  • git status (перевірка стану репозиторію)
  • git add (додати всі файли)
  • git commit (внесення змін)
  • git log (перегляд історії коммітів)
  • git checkout (відміна підготовлених змін)
  • git branch (перегляд списку гілок)
  • git push (надсилання змін у віддалений репозиторій)
  • git pull (отримання змін із віддаленого репозиторію)
  • git merge (злиття віддаленого репозиторію з локальним)
  • git clone (клонування репозиторію)

.gitignore потрібен для приховання файлів та папок від системи контролю версій Git. Зазвичай приховують файли конфігурації (особливо з паролями), тимчасові файли та папки.

Advanced

  • git revert (використовується для відкату змін історії коммітів репозиторію)
  • git reset (складний універсальний інструмент для скасування змін)
  • git rebase (спосіб перенести зміни з однієї гілки до іншої)
  • git cherry-pick (бере зміни, що вносяться одним коммітом, і намагається повторно застосувати їх у вигляді нового комміту в поточній гілці)
  • git stash (дозволяє на якийсь час «здати в архів» (або відкласти) зміни, зроблені в робочій копії, щоб ви могли застосувати їх пізніше)
